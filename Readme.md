# AI Startup Investment Evaluator

> LangGraph 기반 멀티 에이전트 시스템으로 스타트업 투자 평가를 자동화하는 프로젝트

## 📋 프로젝트 개요

벤처캐피탈(VC) 투자 심사 과정에서 수많은 스타트업을 평가하고 보고서를 작성하는 작업은 시간이 많이 소요되고 반복적인 업무가 많습니다. 

이 프로젝트는 플랫폼에서 스타트업 정보를 자동으로 수집하고, 시장성·경쟁사·기술력 등을 종합 분석하여 투자 적합성을 판단한 후 전문가 수준의 평가 보고서를 생성하는 시스템입니다.

## 🎯 문제 정의

### 비즈니스 문제

- **수동 데이터 수집의 비효율성**: 플랫폼에서 스타트업 정보를 직접 수집하고 정리하는 데 상당한 시간 소요
- **일관성 있는 평가의 어려움**: 평가 기준이 명확하더라도 평가자마다 해석이 달라 일관성 유지 어려움
- **보고서 작성 시간**: 평가 결과를 바탕으로 전문가 수준의 보고서를 작성하는 데 상당한 시간 필요
- **최신 정보 반영**: 산업 동향, 경쟁사 정보 등 최신 데이터를 실시간으로 반영하기 어려움

### 기술적 도전 과제

- 여러 에이전트가 생성한 정보를 보고서 생성 시 누락되는 문제
- 산업 분석 → 기업 분석 → 평가의 논리적 연결이 어색한 문제
- 기업 태그 기반 경쟁사 비교 시 데이터 일관성 부족

## 💡 솔루션 접근 방식

### 아키텍처 선택: LangGraph 기반 멀티 에이전트 시스템

LangGraph를 선택한 이유는 복잡한 비즈니스 로직을 노드 단위로 분리하여 유지보수성을 높이고, 상태 기반 그래프로 동적 워크플로우를 구현할 수 있기 때문입니다.

### 핵심 설계 원칙

1. **상태 기반 데이터 전달**: TypedDict 기반 AppState로 노드 간 데이터 전달 구조 명확화
2. **RAG 기반 정보 검색**: Chroma VectorDB에 기업/산업 데이터 임베딩 저장, 유사도 검색으로 컨텍스트 확보
3. **하이브리드 접근**: 구조화가 중요한 작업은 직접 LLM 호출, 컨텍스트 확보가 중요한 작업은 RAG 활용
4. **조건부 라우팅**: 투자 거부 시 다음 기업으로 자동 전환하는 순환 그래프 구조

### 시스템 흐름

```
스타트업 수집 → 산업 동향 수집 → 기업별 순차 분석
    ↓
시장성 평가 → 경쟁사 분석 → 투자 의사결정
    ↓
보고서 생성 (승인 시) 또는 다음 기업으로 이동 (거부 시)
```

## 🛠 기술 스택

### 핵심 프레임워크
- **LangGraph**: 워크플로우 오케스트레이션 및 노드 간 상태 관리
- **LangChain**: LLM 통합 및 RAG 파이프라인 구축
- **OpenAI GPT-4o-mini**: 평가 및 보고서 생성

### 인프라 및 데이터베이스
- **Chroma VectorDB**: 벡터 데이터베이스 (기업/산업 데이터 임베딩 저장, 메타데이터: type, tags)
- **RDB (관계형 데이터베이스)**: 기업 태그 정보 저장 및 태그 기반 경쟁사 조회
- **HuggingFace Embeddings (BAAI/bge-m3)**: 텍스트 임베딩 생성

### 도구 및 라이브러리
- **Playwright**: 플랫폼 웹 크롤링
- **한경 컨센서스**: 산업 동향 PDF 자료 수집
- **Tavily Web Search API**: 산업 동향 실시간 수집
- **ReportLab**: PDF 보고서 생성
- **Matplotlib**: 레이더 차트 시각화

## 🎯 주요 기능

### 1. 실시간 스타트업 발굴
- 플랫폼에서 Playwright 기반 크롤링으로 최신 스타트업 정보 자동 수집
- 2단계 크롤링: 초기 50개 크롤링 → 태깅 → RDB 저장 → 태그별 선정 → 최종 10개 선정 → 상세 크롤링
- RDB와 VectorDB의 역할 분리로 효율적인 데이터 관리

### 2. RAG 기반 정보 검색
- Chroma VectorDB에 기업/산업 데이터 임베딩 저장 (메타데이터: type, tags)
- 한경 컨센서스 PDF + Tavily Web Search로 산업 동향 수집
- 유사도 검색 및 메타데이터 필터링으로 컨텍스트 확보 후 LLM에 전달하여 토큰 사용량 절감

### 3. 멀티 에이전트 평가 파이프라인
- **기업 탐색**: 50개 크롤링 → 태깅 → RDB 저장 → 태그별 선정 → 최종 10개 선정 → 상세 크롤링 → VDB 저장
- **산업 탐색**: 한경 컨센서스 PDF + Tavily Web Search로 산업 동향 수집 및 VDB 저장
- **시장성 평가**: VDB에서 산업/기업 정보 조회 후 LLM으로 시장성 평가 보고서 생성
- **경쟁사 분석**: RDB에서 태그 기반 경쟁사 조회 후 LLM으로 경쟁 구도·차별점·위험요소 분석
- **투자 여부 판단**: 항목별 0/1 판단을 종합하여 투자 여부 True/False 결정
- **보고서 작성**: State 조합 + 필요 시 VDB 추가 검색하여 PDF 보고서 생성

### 4. 자동 보고서 생성
- 투자 승인 기업 대상 상세 투자 보고서 작성
- 레이더 차트 시각화 (평가 항목별 점수)
- 전체 거부 시 종합 거부 사유 분석 보고서 작성

## 📊 시스템 요약

이 시스템은 **6개의 주요 에이전트**가 협업하여 스타트업 투자 평가를 자동화합니다. 각 에이전트는 특정 역할을 담당하며, LangGraph의 상태 관리 시스템을 통해 정보를 공유하고 전달합니다.

핵심은 **상태 기반 데이터 전달**과 **조건부 라우팅**을 통해 여러 기업을 순차적으로 평가하고, 투자 승인 기업에 대해서만 상세 보고서를 생성하는 것입니다.

자세한 아키텍처와 파이프라인 흐름은 [docs/architecture.md](docs/architecture.md)와 [docs/pipeline.md](docs/pipeline.md)를 참고하세요.

## 🔧 트러블슈팅 요약

### 1. 여러 Agent가 생성한 정보 누락 문제
**문제**: 보고서 생성 시 State에 저장된 정보가 일부 누락됨  
**해결**: State key 표준화 및 명시적 데이터 검증 로직 추가  
**결과**: 모든 에이전트의 분석 결과가 보고서에 반영됨

### 2. 논리적 연결성 부족 문제
**문제**: 산업 분석 → 기업 분석 → 평가의 논리적 연결이 어색함  
**해결**: 구조화된 보고서 템플릿 설계 및 프롬프트 엔지니어링  
**결과**: 자연스러운 흐름의 전문가 수준 보고서 생성

### 3. 데이터 일관성 부족 문제
**문제**: 기업 태그 기반 경쟁사 비교 시 데이터 일관성 부족  
**해결**: 태그 그룹 단위 비교 방식으로 개선  
**결과**: 경쟁사 분석의 정확도 및 일관성 향상

자세한 트러블슈팅 과정은 [docs/troubleshooting.md](docs/troubleshooting.md)를 참고하세요.

## 🎓 배운 점

- **LangGraph 워크플로우 설계**: 상태 기반 그래프로 복잡한 비즈니스 로직을 노드 단위로 분리하여 유지보수성 향상
- **RAG 파이프라인 구축**: 임베딩 모델 선택, 청킹 전략, 메타데이터 필터링 등 실제 RAG 시스템 구축 경험

- **프롬프트 엔지니어링**: 구조화된 출력(Structured Output) 활용으로 JSON 파싱 오류 최소화, 평가 점수 일관성 확보
